# Section Library

Common page types and their recommended section architectures. These patterns are drawn from high-converting websites across SaaS, agency, portfolio, and startup verticals. Use these as the default structure when generating section plans — adapt based on the user's specific idea.

---

## Page Type: SaaS / AI Product Landing Page

Best for: Funded startups, AI tools, B2B software, developer tools

```
1. nav-bar (light)          — Logo | Nav links (Product, Pricing, Docs, Blog) | CTA button
2. hero-split (dark)        — Headline (value prop) | Subheadline | CTA | Product screenshot
3. logo-bar (light)         — "Trusted by" + 4-6 company logos (social proof, early)
4. features-cards (light)   — 2-3 pain points in cards or columns. Sets up the "why"
5. features-alternating (light) — Alternating left-right feature blocks with visual + text
6. how-it-works (light)     — 3-step numbered horizontal flow (simple, clear)
7. metrics-bar (dark)       — Big numbers: "10x faster", "50% cost reduction", stats bar
8. testimonials-cards (light) — 1-2 customer quotes with avatar, name, company, role
9. pricing-cards (light)    — 2-3 tier cards (optional — some landing pages skip this)
10. faq-accordion (light)   — Accordion, 4-6 questions
11. cta-banner (dark)       — Centered headline + CTA button + optional subtext
12. footer (dark)           — Logo | Nav columns | Social links | Legal links | Copyright
```

**Variants:**
- If pre-launch / waitlist: Replace Pricing with an email capture form
- If developer-focused: Add a code snippet / terminal block in the Hero
- If enterprise: Add a "Security & Compliance" section before Pricing

## Page Type: Agency / Studio Portfolio

Best for: Design studios, dev agencies, creative firms

```
1. Navigation        — Logo | Nav (Work, Services, About, Contact) | CTA
2. Hero              — Bold statement or tagline | Brief descriptor | CTA or scroll prompt
3. Selected Work     — 2-3 project cards: image + project name + client + brief result
4. Services          — 3-4 service blocks: title + short description
5. Process           — 3-4 step horizontal or vertical flow showing how you work
6. Clients/Logos     — Logo bar of past clients
7. Testimonial       — Single featured quote
8. About/Team        — Short studio bio + optional team photo or headshots
9. CTA Section       — "Let's work together" + contact CTA
10. Footer           — Logo | Links | Social | Copyright
```

**Variants:**
- If solo freelancer: Replace Team with a personal bio section
- If case-study focused: Make Selected Work larger with more detail per project
- If niche-specific: Add a "Who We Work With" section describing ideal client

## Page Type: Web App Dashboard

Best for: SaaS products, internal tools, admin panels

```
1. Top Bar           — Logo | Search | Notifications icon | User avatar + dropdown
2. Sidebar           — Navigation links with icons | Collapsible sections | Active state
3. Page Header       — Page title | Breadcrumb | Action buttons (Add, Filter, Export)
4. Stats/KPI Row     — 3-4 metric cards (number + label + trend indicator)
5. Primary Content   — Data table OR chart/graph area (main workspace)
6. Secondary Content — Side panel or below-fold: activity feed, recent items, quick actions
7. Empty State       — Placeholder for when no data exists (illustration + CTA)
```

**Variants:**
- If analytics-heavy: Replace Primary Content with a chart grid (2x2)
- If task-management: Use kanban columns instead of data table
- If settings page: Form-based layout with grouped input sections

## Page Type: Product / Feature Page

Best for: Specific feature deep-dives, product detail pages

```
1. Navigation        — Same as main site nav
2. Feature Hero      — Feature name + description + screenshot/demo visual
3. Problem           — What pain this feature solves (short, 1-2 sentences + visual)
4. Capability Grid   — 3-4 sub-features in a grid with icon + title + description
5. Demo/Screenshot   — Large product screenshot or embedded demo placeholder
6. Comparison        — Before/after or "without us vs. with us" table
7. Integration       — "Works with" logo bar or integration cards
8. Testimonial       — Feature-specific customer quote
9. CTA               — Try it / Get started
10. Footer           — Standard footer
```

## Page Type: Pricing Page

Best for: SaaS pricing, service packages

```
1. Navigation        — Standard nav
2. Pricing Header    — Page title + short descriptor ("Simple, transparent pricing")
3. Toggle            — Monthly / Annual switch (if applicable)
4. Pricing Cards     — 2-3 tier cards: name, price, feature list, CTA button. Highlight recommended tier.
5. Feature Compare   — Full comparison table (rows = features, cols = tiers, checkmarks)
6. FAQ               — Pricing-specific questions (billing, refunds, upgrades)
7. Enterprise CTA    — "Need custom?" + Contact sales CTA
8. Footer            — Standard footer
```

## Page Type: About Page

Best for: Company story, team, values

```
1. Navigation        — Standard nav
2. Hero              — Company mission statement or bold "about" headline
3. Story             — 2-3 paragraphs + timeline or key milestones
4. Values/Principles — 3-4 value cards with icon + title + description
5. Team              — Grid of headshots + name + role (or single founder bio)
6. Numbers           — Key stats: team size, clients served, years in business
7. CTA               — "Join us" or "Work with us"
8. Footer            — Standard footer
```

## Page Type: Blog / Content Hub

Best for: Content marketing, resource centers

```
1. Navigation        — Standard nav with Blog highlighted
2. Featured Post     — Large card: image + title + excerpt + author + date
3. Category Filter   — Horizontal tag/category bar
4. Post Grid         — 3-column grid of post cards: image + title + excerpt + date
5. Pagination        — Page numbers or "Load more" button
6. Newsletter CTA    — Email capture: "Subscribe for updates" + email input + submit
7. Footer            — Standard footer
```

## Page Type: Contact Page

Best for: Lead capture, inquiry forms

```
1. Navigation        — Standard nav
2. Contact Header    — "Get in touch" + short descriptor
3. Two-Column Layout — Left: contact form (name, email, message, submit) | Right: contact info (email, phone, address, map placeholder)
4. Office/Location   — Map placeholder + address details
5. FAQ               — Common pre-sales questions
6. Footer            — Standard footer
```

---

## Page Type: Login / Auth Screen

Best for: Sign in, sign up, onboarding

```
1. Split Layout      — Left: brand visual / illustration placeholder | Right: auth form
2. Auth Form         — Logo + heading + email input + password input + submit button + "Forgot password?" link + "Sign up" link
3. Social Auth       — Divider ("or") + Google/GitHub/SSO buttons
4. Footer            — Minimal: links to Terms, Privacy
```

## Universal Sections (mix into any page)

These can be inserted into any page type where relevant:

- **Announcement Bar** — Top of page, full-width, dismissible. "New: We just launched X →"
- **Logo Bar** — Company logos for social proof. Works after Hero or before Footer.
- **Testimonials (Single)** — Quote + avatar + name + role + company
- **Testimonials (Grid)** — 2-3 testimonial cards in a row
- **Testimonials (Carousel)** — Sliding testimonial cards
- **CTA Banner** — Full-width colored band with headline + CTA button
- **Newsletter Signup** — Email input + submit button, minimal
- **FAQ Accordion** — Question + expandable answer, 4-8 items
- **Stats Bar** — 3-4 big numbers with labels in a row
- **Integration Logos** — "Works with" + logo grid
- **Comparison Table** — Feature comparison with checkmarks across columns
- **Timeline** — Vertical or horizontal steps with dates/milestones
- **Video Embed** — Video placeholder with play button overlay

## Section Composition Rules

When building a section plan:

1. **Hero is always second** (after Navigation). No exceptions.
2. **Social proof early** — Logo bar or metrics within the first 3 sections after Hero.
3. **Don't stack similar sections** — No two feature grids back-to-back. Alternate visual patterns (grid → alternating → single column).
4. **CTA rhythm** — Include a CTA every 3-4 sections. Don't make the user scroll to the bottom to take action.
5. **Footer is always last.** Navigation is always first.
6. **Maximum 12-14 sections** for a single-page site. If more, suggest multi-page.
7. **Minimum 6 sections** for a credible landing page (Nav, Hero, Features, Social Proof, CTA, Footer).
8. **Testimonials before Pricing** — Social proof should precede the ask.
9. **FAQ before final CTA** — Address objections, then ask for the conversion.
